이 사이트는?

Jindo 라이브리러리에 익숙한 개발자를 대상으로 jQuery 라이브러리 마이그레이션에 도움을 주기 위해 만들어졌습니다. 궁금한 부분을 쉽고 빠르게 접근하고 모든 개발자들이 Jindo 에서 jQuery 로 이전 하면서 쌓인 노하우를 쉽게 공유함으로써 지속적으로 양질의 코드를 양산할 수 있는 성장형 웹 코드 커뮤니티를 지향합니다.

어떤 jQuery 메소드를 써야할지를 해결해드립니다

이제 막 Jindo 를 마스터하고 우쭐우쭐하고 있었지만 존도 쓰고 제임스도 쓰다가 이제는 우리팀 찬욱님, 희재님도 jQuery 를 한다고 하니 이제 나 혼자만 Jindo 를 쓸 수 없을 것만 같습니다. 근데 막상 jQuery 문서를 보면 어디서 부터 봐야할 지 막막합니다. 지금 다른 일도 할게 많은데 언제 다 볼까...하며 깊은 시름에 빠진 분들을 위한 맞춤형 서비스 입니다. 익숙하게 쓰던 Jindo API 를 중심으로 그에 매핑되는 jQuery 변환 방법을 제공.. 아니 공유하는 장 입니다. <스샷.!!>

혹시 더 좋은 코드 알고 계신가요? 그럼 공유해주세요.

우리는 Jindo 마스터들이지만 아직은 jQuery 코드에 서툽니다. 물론 Jindo 를 마스터했으니 jQuery 도 곧 마스터할 겁니다. 하지만 언제나 성장과 배움은 시행착오와 삽질을 동반합니다. "아...이 산이 아닌가부다!" 라고 하기에는 우린 각자 너무 많은 짐들을 짊어지고 있습니다. 우리 똑같은 문제로 모두 똑같이 너무 힘들어하지 말고 사랑하는 동료들과 나눠서 조금만 힘들었으면 좋겠습니다. <댓글을 통한 의견/코드 전달 스샷> 우연히 혹은 정말 너무 힘들게 얻었던 경험이나 코드들을 아래와 같이 댓글로 공유해 주신다면 jQuery 마이그레이션으로 어려움을 겪는 동료들이 똑같은 시행착오를 거치지 않고 빠르게 학습할 수 있는 경험을 얻을 수 있을 것 입니다. 코드는 내부 심사를 거친 후 바로 사이트에 반영됩니다. 선정을 통해 푸짐한 상금과 상품을 드리지는 못하겠지만 "명예의 전당"을 만들어 실력자로 인정받을 수 있는 자리를 만들 계획을 갖고 있습니다.

초보자의 길을 안내하는 것에서부터 고퀄리티 코드 공유 까지

처음에는 대부분 Jindo 에서 jQuery 로의 1 대 1 관계의 코드들이 선보이겠지만, 우리들도 점점 성장할 것이고 쌓이고 쌓인 경험들과 노하우를 통해. 같은 기능이지만 전보다 더 좋은 성능과 보기 좋은 코드를 만들어낼 수 있을 것입니다. 단언컨데 어제나 오늘 보다 나은 내일을 만들어 갈 수 있을 것입니다.
